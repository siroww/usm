1.
Колбэки позволяют писать декларативный код, фокусируясь на результате, 
а не на процессе перебора. Это делает логику обработки данных гибкой, компактной и легко переиспользуемой.

2.
Основными проблемами являются запутанная вложенность («ад колбэков») 
и потеря контекста this. Для их решения используют промисы, async/await 
и стрелочные функции.

3.
Каждый метод можно воссоздать через цикл for
который последовательно вызывает колбэк для каждого элемента. 
Внутри цикла результат либо накапливается в новый массив (map, filter),
либо возвращается сразу при выполнении условия (find, some).
