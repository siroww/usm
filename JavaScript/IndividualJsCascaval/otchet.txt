избраное находятся в локалхосте, А также загружается туда и все 10 монет чтобы небыло проблем с (недоступом к сайту)

ПРОБЛЕМЫ
сам сайт не дает обновить страницу больше 1 раза, так как там стоит блок 

Чтобы обойти блокировку, применяется кеширование ответов API и ограничение числа запросов.
Решение: сохранять данные в localStorage и обновлять их не чаще, чем раз в 1 минуту.

Также реализован механизм повторной загрузки при ошибках сети с экспоненциальной задержкой.
при отключенном интернете приложение корректно использует кешированные данные.

Добавлены сообщения для пользователя о состоянии загрузки и ошибках.

Интерфейс адаптивен и корректно отображается на мобильных устройствах.

Производительность оптимизирована за счёт дебаунса для поля поиска.

Фильтрация и сортировка выполняются на стороне клиента для быстрого отклика.

Код разделён на модули: render.js, storage.js, api.js.
Добавлена возможность отмечать монеты как избранные и фильтровать по ним.

Избранные монеты остаются при перезагрузке благодаря localStorage.

В разделе настроек можно выбрать валюту отображения (USD, EUR и т.д.).

Проект сопровождается простым набором тестов и инструкцией по запуску.

Документация включает примеры использования основных функций.

Были использованы современные методы работы с асинхронностью (async/await).

Ограничения: приложение зависит от доступности API CoinGecko.

Заключение: приложение удовлетворяет требованиям индивидуальной работы и демонстрирует практические навыки.
