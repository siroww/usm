Отчёт по Python/lab7

1. Назначение проекта:
Проект представляет собой веб-приложение на Flask для работы с личной библиотекой.
Приложение позволяет хранить, просматривать и анализировать данные по книгам.

2. Структура проекта:
- app.py — основной файл приложения Flask.
- requirements.txt — список зависимостей для Python.
- database.db — SQLite-база данных с таблицей books.
- templates/ — HTML-шаблоны для отображения страниц.
- static/ — статические файлы (CSS, изображения, если есть).

3. Основные функции:
- Запуск сервера Flask из файла app.py.
- Инициализация базы данных и создание таблицы books.
- Заполнение базы данных из стартового списка книг при первом запуске.
- Просмотр главной страницы с последними добавленными книгами.
- Просмотр списка всех книг с фильтрацией и сортировкой.
- Открытие детальной страницы книги по идентификатору.
- Добавление новой книги через форму с валидацией данных.
- Редактирование существующей книги.
- Удаление книги из базы данных.
- Страница статистики с подсчетом средних значений и топ-5 книг.

4. Модель данных:
Таблица books содержит поля:
- id (INTEGER PRIMARY KEY AUTOINCREMENT)
- title (TEXT)
- author (TEXT)
- genre (TEXT)
- pages (INTEGER)
- rating (REAL)
- created_at (TEXT)

5. Особенности реализации:
- Используется sqlite3 для работы с базой данных.
- row_factory настроен на sqlite3.Row для удобного доступа к полям.
- Валидация данных для полей title, author, genre, pages, rating.
- Поддержка сортировки списка книг по нескольким столбцам.
- Формирование списка жанров динамически из базы данных.
- Использование flash-сообщений для успеха и ошибок.

6. Маршруты приложения:
- / — главная страница с последними книгами.
- /books — список книг.
- /books/<int:book_id> — детальная страница книги.
- /add — добавление книги.
- /books/<int:book_id>/edit — редактирование книги.
- /books/<int:book_id>/delete — удаление книги.
- /stats — страница статистики.

7. Запуск проекта:
1. Создать виртуальное окружение и активировать его.
2. Установить зависимости из requirements.txt.
3. Запустить файл app.py.
4. Открыть браузер и перейти на http://127.0.0.1:5000.

8. Рекомендации:
- Проверить наличие файла database.db в корне проекта.
- При необходимости удалить базу и перезапустить для сброса данных.
- Доработать шаблоны для улучшения интерфейса.
- Добавить обработку ошибок и страницы 404.

9. Итог:
Проект содержит все базовые функции для персональной библиотеки.
Он удобен для демонстрации навыков работы с Flask и SQLite.
