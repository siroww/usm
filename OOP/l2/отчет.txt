Отчет по лабораторной работе: Иерархия «Сторона – Квадрат – Куб»

Описание структуры программы
Программа реализует иерархию классов, используя принципы объектно-ориентированного программирования (наследование и полиморфизм):

Класс Side (Сторона): Фундамент иерархии. Хранит базовые данные: название объекта и длину его стороны (a).

Класс Square (Квадрат): Наследует свойства Стороны. Содержит метод для вычисления площади поверхности по формуле S = a * a.

Класс Cube (Куб): Наследует свойства Квадрата. Вводит новую характеристику — толщину стенок. Это позволяет моделировать реальный полый объект.

Логика вычислений
Основная сложность задачи заключалась в расчете полезного объема:
Учет стенок: Чтобы найти пространство внутри куба, мы вычитаем толщину стенок с обеих сторон. Формула внутреннего ребра: a_vnutr = a - (2 * толщина).
Объем: Вычисляется как куб внутренней стороны. Если стенки настолько толстые, что смыкаются, программа выдает объем 0.
Сравнение: В методе main программа сравнивает объемы двух кубов, находит среди них меньший и делит объем большого на объем малого. Это позволяет узнать, сколько «порций» воды из маленького куба влезет в большой.

Ключевые особенности реализации
Конструкторы: Классы Square и Cube имеют собственные конструкторы для инициализации параметров при создании объектов.
Полиморфизм: Использование виртуальной функции show() позволяет выводить полную информацию об объекте (название, сторона, площадь, толщина, объем) одной командой.
Безопасность: Добавлена проверка на отрицательный объем (если стенки толще самого куба).

Результат выполнения
Программа успешно создает два куба, выводит их полные характеристики в консоль, определяет меньший из них и рассчитывает количество воды для заполнения большего куба. Текст вывода в консоли реализован на транслите для исключения проблем с кодировкой в терминале.
